Laboratório Completo de Formulários HTML5

Este documento contém todos os tipos de campos e elementos de formulário nativos do HTML, acompanhados de explicações conceituais sobre seus comportamentos e validações automáticas do navegador.

5. Elementos de Seleção

Múltipla Escolha (type="checkbox")

Conceito: Múltiplas caixas podem ser marcadas. Ideal para "Selecione as tecnologias que você domina".

Escolha Única (type="radio")

Conceito: Quando vários inputs do tipo radio possuem o mesmo atributo 'name', eles formam um grupo onde apenas um pode ser selecionado por vez. Conceito: A tag optgroup agrupa as opções (options) logicamente, criando cabeçalhos não-clicáveis dentro da lista suspensa. Conceito: Diferente do <select>, onde você é obrigado a escolher o que está na lista, o <datalist> acoplado a um input permite que o usuário aceite a sugestão ou digite algo completamente novo.
6. Upload e Seleção de Cor Conceito: Abre o explorador de arquivos do sistema. O 'accept' filtra o que aparece, e 'multiple' deixa enviar mais de um arquivo de uma vez. Note que o formulário precisa ter o atributo enctype="multipart/form-data" para enviar os arquivos reais para o backend. Conceito: Abre a paleta de cores nativa do sistema operacional, retornando o valor em formato hexadecimal (#RRGGBB).
7. Campos Ocultos e Ações (Botões)

Campo Oculto (type="hidden"):

Conceito: Este campo está no HTML abaixo, mas é invisível. Serve para armazenar dados (como o ID do usuário ou um token de segurança) que precisam ser enviados ao backend sem que o usuário os veja ou altere diretamente na tela.

Tipos de Botões:

Conceito: O type="reset" limpa o formulário de volta aos valores originais. O type="submit" tenta disparar a "action" do formulário após checar as validações. A tag <button> é mais moderna e flexível que o <input type="button">, pois permite colocar imagens e ícones dentro dela.

Botão de Imagem (type="image"):

Conceito: Funciona exatamente como um submit, mas usa uma imagem como botão. Pouco utilizado hoje em dia devido ao CSS, mas faz parte da especificação. (O alt aparece porque não há imagem no src).
