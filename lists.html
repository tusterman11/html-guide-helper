Laboratório: Estruturas de Listas em HTML

Este documento demonstra os diferentes tipos de listas nativas do HTML e seus atributos de formatação estrutural.

1. Listas Não Ordenadas (<ul>)

Utilizadas quando a ordem dos itens não altera o sentido ou a compreensão do conteúdo.

  • Configuração de Ambiente
  • Instalação de Dependências
    • Bibliotecas de Frontend
    • Módulos de Backend
  • Configuração de Variáveis Globais

2. Listas Ordenadas (<ol>)

Utilizadas quando a sequência dos itens é estritamente importante, como em um passo a passo, ranking ou cronograma.

Ordem Padrão (Numérica)

  1. Levantamento de Requisitos
  2. Modelagem de Dados
  3. Desenvolvimento da Interface

Atributo: start (Iniciando de um valor específico)

Adicionando o atributo start="5" na tag da lista:

  1. Testes Unitários
  2. Homologação com o Cliente

Atributo: reversed (Contagem regressiva)

Adicionando o atributo booleano reversed na tag da lista:

  1. Revisão Final de Código
  2. Backup do Banco de Dados
  3. Migração para Produção

Atributo: type (Formatos de numeração alternativos)

Letras maiúsculas usando type="A":

  1. Ambiente de Desenvolvimento
  2. Ambiente de Testes
  3. Ambiente de Produção

Algarismos Romanos usando type="I":

  1. Módulo de Autenticação
  2. Módulo de Pagamentos

Atributo na tag do item: value

A tag <li> pode receber o atributo value para forçar um salto na numeração. O item abaixo recebeu value="15":

  1. Passo inicial (1)
  2. Passo seguinte (2)
  3. Passo com salto manual (15)
  4. Passo automático (16)

3. Listas de Definição ou Descrição (<dl>)

Estrutura utilizada para agrupar pares de termos e suas respectivas descrições, funcionando de maneira semelhante a um dicionário ou glossário de conceitos.

<dt> (Definition Term)
Define o termo, nome ou conceito que será explicado. Fica alinhado à esquerda por padrão.
<dd> (Definition Description)
Contém a explicação ou os detalhes do termo acima. O navegador aplica nativamente um recuo (indentação) à esquerda para diferenciá-lo do termo principal.
Front-end
A camada da aplicação com a qual o usuário interage diretamente (interface gráfica).
Back-end
A camada da aplicação responsável pelas regras de negócio, processamento de dados e comunicação com o banco de dados.
